## üéØ Visi√≥n General

El sistema de trabajos permite que los **clientes** publiquen trabajos y los **profesionales** apliquen a ellos. El flujo garantiza que:

- ‚úÖ Solo profesionales pueden aplicar a trabajos
- ‚úÖ La conversaci√≥n requiere haber aplicado primero
- ‚úÖ Solo el cliente puede aceptar al profesional
- ‚úÖ La ubicaci√≥n exacta solo se revela al profesional aceptado
- ‚úÖ El trabajo se cierra autom√°ticamente al aceptar un profesional
- ‚úÖ Se crea un evento en el calendario autom√°ticamente

  -- Referencia al trabajo original (opcional, para trazabilidad)

## üìã Casos de Uso

### Caso 1: Flujo Exitoso

```
1. Cliente crea trabajo ‚Üí status: 'open'
2. Profesional A ve trabajo ‚Üí ubicaci√≥n aproximada
3. Profesional A aplica ‚Üí ve ubicaci√≥n exacta
4. Profesional A inicia chat ‚Üí conversaci√≥n vinculada al trabajo
5. Cliente acepta a Profesional A ‚Üí
   - status cambia a 'in_progress'
   - selected_professional_id = Profesional A
   - Se crea evento en calendario
   - Se rechazan otras aplicaciones
   - Profesional A ve botones de navegaci√≥n
6. Trabajo desaparece de b√∫squedas p√∫blicas
7. Profesional B (no aceptado) pierde acceso a ubicaci√≥n exacta
```

### Caso 2: Cliente Cancela Trabajo

```
1. Cliente crea trabajo ‚Üí status: 'open'
2. Varios profesionales aplican
3. Cliente cancela trabajo ‚Üí status: 'canceled'
4. Trabajo desaparece de b√∫squedas
5. Aplicaciones se marcan como rechazadas
6. Se notifica a profesionales aplicados
```

### Caso 3: Trabajo Expira

```
1. Cliente crea trabajo ‚Üí expires_at: 30 d√≠as
2. No recibe aplicaciones
3. Trigger autom√°tico ‚Üí status: 'expired'
4. Trabajo se oculta de b√∫squedas
```
